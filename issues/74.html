History

Updated by Jeremy Wright about 10 years ago

J, can you try hitting the Enter key after changing the serial port to see if it updates? I forgot that hitting Enter was a requirement to trigger the change event. This needs to be altered if for no other reason than it wasn't intuitive. A drop down list showing the available serial ports would most likely be the best way to handle this.

#2
Updated by Jeremy Wright almost 10 years ago

  • Status changed from In Progress to Resolved
  • % Done changed from 0 to 100

Changed the serial port text field to a drop down list. The list is populated with the serial ports that are currently available on the system, which on newer systems will probably just be the Arduino, and the first available is selected by default when the program starts up. When the serial port is changed the connection should pick back up where it left off.
